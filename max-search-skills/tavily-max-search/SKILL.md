# Tavily Max Search — 并行搜索执行器

基于预定义的搜索关键词，并行调用 Tavily API 并格式化结果为可分析的上下文。

## 前置要求

- Python 3.8+
- Tavily API Key（首次使用需配置）
- 可选：`pip install aiohttp`（提升并行性能，不可用时自动回退到同步模式）

## 配置 API Key

### 首次使用检查

运行以下命令查看配置状态：

```bash
python scripts/search.py config
```

如果未配置，脚本会输出详细的配置引导。

### 保存 API Key（推荐方式）

```bash
python scripts/search.py config --set-api-key YOUR_TAVILY_API_KEY
```

密钥将保存到 `~/.tavily_api_key`（文件权限 0600），后续自动加载。

### 临时使用（环境变量）

```bash
export TAVILY_API_KEY="YOUR_TAVILY_API_KEY"
```

**加载优先级**：环境变量 > `~/.tavily_api_key` 文件

**获取 API Key**：访问 <https://app.tavily.com/home>

## 使用方式

### 输入参数

- `--question`：用户的原始问题（用于上下文标注）
- `--search-json`：搜索配置 JSON（必须包含 `search_queries` 和 `num_results`）

### 调用命令

```bash
python scripts/search.py search \
  --question "用户的原始问题" \
  --search-json '{"search_queries":["关键词1","关键词2"],"num_results":7}'
```

### 输出内容

脚本输出格式化的搜索结果摘要（Markdown），包含：

1. **搜索概况统计**
   - 执行的查询数 / 成功数
   - 总条目数
   - 高相关度条目数（score ≥ 0.8）

2. **分查询的结果列表**
   - 每个关键词对应的搜索结果
   - 每条结果包含：标题、URL、内容摘要、相关度评分
   - Tavily AI Answer（如果可用）

3. **当前时间戳**（UTC）
   - 用于判断"最近"、"今年"等时间词的具体含义

## 回答用户时的原则

基于脚本输出的搜索结果回答用户问题时，必须遵循：

### 1. 事实基准（Grounding）

- **搜索结果优先于内部知识**：时效性信息有冲突时，以搜索结果为准
- **诚实标注空白**：搜索结果未覆盖的信息要明确说明"搜索结果未提及"
- **严禁编造数据**：没有找到的信息不要猜测

### 2. 综合叙述

- **不要简单罗列**：交叉验证不同来源，综合分析
- **对比验证**：如果多个来源说法不一致，指出差异并说明可能原因
- **结构化呈现**：使用标题、列表、表格提升可读性

### 3. 引用来源

- **提供真实可点击的链接**：格式 `[标题](URL)`
- **标注来源可信度**：如果某来源评分较低，可适当说明

## 完整工作流示例

假设用户问："打工人最近流行什么减压方式？"

### Step 1 — 先用 `tavily-keyword-extractor` 生成搜索配置

输出：

```json
{
  "refined_question": "职场人士当前主流的压力管理方法和减压活动有哪些？",
  "search_queries": [
    "职场人士 减压方式 2026",
    "上班族 压力管理 流行趋势",
    "工作压力 缓解方法 调查报告"
  ],
  "num_results": 7,
  "reasoning": "用户说的'打工人'是网络用语（指职场人士/上班族），拆解为当前趋势、管理方法、调查数据三个维度"
}
```

### Step 2 — 用本 Skill 执行搜索

```bash
python scripts/search.py search \
  --question "打工人最近流行什么减压方式？" \
  --search-json '{"search_queries":["职场人士 减压方式 2026","上班族 压力管理 流行趋势","工作压力 缓解方法 调查报告"],"num_results":7}'
```

### Step 3 — 基于脚本输出回答用户

脚本会输出完整的搜索结果摘要，你需要：

1. 阅读所有搜索结果
2. 提取关键信息（流行减压方式、趋势变化、专家建议）
3. 综合叙述，避免简单复制粘贴
4. 在回答末尾列出参考来源

## 特殊情况处理

### 情况 1：search\_queries 为空

如果输入的 JSON 中 `search_queries` 为空数组，脚本会输出：

```
No web search was performed (the query was classified as not requiring 
external information). Please answer the following question directly 
using your own knowledge:

<用户问题>
```

此时直接用内部知识回答，无需搜索。

### 情况 2：API Key 未配置

脚本会输出详细的配置引导并退出（exit code 1）。你需要：

1. 告知用户需要配置 Tavily API Key
2. 提供配置命令：`python scripts/search.py config --set-api-key YOUR_KEY`
3. 引导用户访问 <https://app.tavily.com/home> 获取密钥

### 情况 3：所有搜索失败

脚本会输出错误信息并退出（exit code 1）。可能原因：

- API Key 无效
- 网络连接问题
- Tavily 服务故障

建议用户检查 API Key 和网络连接。

## 技术细节

- **并行执行**：如果安装了 `aiohttp`，所有查询并行发送；否则使用线程池（`ThreadPoolExecutor`）
- **域名过滤**：自动排除低质量域名（阴谋论网站、小报、极端政治媒体等）
- **超时控制**：单个查询超时 30 秒
- **错误处理**：单个查询失败不影响其他查询，最终只要有一个成功即可继续

## 注意事项

- 单次搜索总条目数建议控制在 40 以内，避免 context 过载
- 脚本的 stderr 输出包含执行日志（可忽略），stdout 输出才是搜索结果
- 如果需要更新 API Key，重新运行 `config --set-api-key` 即可覆盖
